# NetFabric

Network management and automation platform for MSPs and network engineers. Connect to your Cisco (and other) devices over SSH, run AI-driven analysis using Claude, and manage configurations — all from a single web UI.

## Features

- **Device management** — add and manage network devices with encrypted credential storage
- **SSH connectivity** — connects via Netmiko/Paramiko, supports legacy SHA1 KEX for older devices
- **AI analysis** — powered by Anthropic Claude; runs BGP, config review, log analysis, and status checks
- **Config snapshots & diffs** — track configuration changes over time
- **JWT authentication** — secure multi-user access
- **REST API** — full OpenAPI docs at `/docs`

## Tech Stack

- **Backend**: FastAPI, SQLAlchemy, PostgreSQL
- **Auth**: JWT (python-jose) + Fernet encryption for device passwords
- **Network**: Netmiko, NAPALM, Paramiko
- **AI**: Anthropic Claude API
- **Frontend**: Vanilla HTML/JS (`frontend/index.html`)

## Getting Started

### Prerequisites

- Docker and Docker Compose

### Run (development)

Generate the required keys:

```bash
openssl rand -hex 32                                              # SECRET_KEY
python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"  # FERNET_KEY
openssl rand -hex 16                                              # POSTGRES_PASSWORD
```

Create `.env` with the generated values, then:

```bash
docker compose up --build
```

Open [http://localhost:8000](http://localhost:8000) for the UI, or [http://localhost:8000/docs](http://localhost:8000/docs) for the API.

## License Management

NetFabric uses a signed license file to enforce node limits. Licenses are generated by the vendor and uploaded by the client via the web UI.

### Tiers

| Tier | Max devices |
|------|-------------|
| `1` | 1 |
| `10` | 10 |
| `100` | 100 |
| `1000` | 1000 |
| `unlimited` | No limit |

### Generate a license

```bash
python3 scripts/generate_license.py <customer_id> <tier>
```

Examples:

```bash
python3 scripts/generate_license.py acme 10
python3 scripts/generate_license.py bigco unlimited
```

The file is saved to `~/Desktop/license_<customer_id>.json`. Send it to the customer — they upload it via the web UI on first login.

### How it works

- `scripts/generate_license.py` — vendor-only tool, never shipped to clients
- `backend/license_guard.py` — compiled to a `.so` binary by Nuitka during `docker build`; the source `.py` is deleted from the final image
- Clients receive only the Docker image — the license logic is opaque and tamper-proof

### Rebuild and ship

After any code change:

```bash
docker buildx build --platform linux/amd64,linux/arm64 \
  -t bashar1aziz/netfabric:latest --push .
```

Clients upgrade with:

```bash
docker compose pull && docker compose up -d
```

## Security Notes

- Device passwords are encrypted at rest using Fernet symmetric encryption
- Never commit `.env` or `license_*.json` — both are in `.gitignore`
- Change `_SECRET_KEY` in `license_guard.py` and `generate_license.py` before production release
